# 결제 자동화 설계 (Payment Automation Design)

세 개의 유료 서비스 — **HR English Coach(Pro)**, **AI 성장 아카이브(멤버십)**, **단청의 책장(reading.html 멤버십)** — 을
정적 사이트 구조를 유지한 채, 결제 → 라이선스 발급 → 앱 잠금 해제까지 **사람 손이 거의 안 타게** 자동화하기 위한 설계입니다.

---

## 1. 문제 정의

현재 상태(수동 MVP):
- 사용자가 mailto로 신청 → 운영자가 계좌 확인 → 라이선스 코드 수동 발송 → 사용자가 코드 입력
- 잠금은 클라이언트 사이드(SHA-256 해시 목록). 유료 사용자가 10~20명일 땐 충분하지만, 늘면 병목.

자동화가 필요한 2가지 지점:
1. **결제** — 카드/간편결제로 즉시 결제
2. **라이선스 발급** — 결제 즉시 코드가 자동 생성·전달

할인은 **연간 구독(33~34% 절약)** 하나로 단순화합니다. (초기엔 명함 인증 할인을 검토했으나,
HR English Coach는 타깃 자체가 HR이라 변별력이 없어 제외했습니다.)

제약: **백엔드 서버를 두지 않는다**(GitHub Pages 정적 호스팅 유지, 운영 부담 최소화).

---

## 2. 추천 스택 — Merchant of Record + 클라이언트 검증

정적 사이트에서 결제 자동화의 핵심 난제는 "결제했는지를 서버 없이 어떻게 확인하나?"입니다.
답: **Merchant of Record(MoR)가 발급하는 라이선스 키를, 그들의 API로 브라우저에서 직접 검증**한다.

### 왜 Merchant of Record인가
- 개인(1인 사업자)도 PG사 심사 없이 바로 판매 가능 — 국내 PG 직접 계약의 가장 큰 진입장벽 해소
- **부가세·해외 세금을 대신 처리**(MoR이 판매 주체) — 세무 리스크 없음
- **구독(정기결제) + 라이선스 키 자동 발급**이 기본 기능
- 결제 실패·환불·카드 만료 갱신 등 구독 라이프사이클 자동 관리

### 후보 비교

| 서비스 | 구독 | 라이선스 키 | 클라이언트 검증 API | 수수료(대략) | 비고 |
|---|---|---|---|---|---|
| **Lemon Squeezy** (추천) | ✅ | ✅ 내장 | ✅ License API (브라우저 POST 가능) | 5% + 결제수수료 | 현재 Stripe 산하, 문서 우수 |
| Paddle | ✅ | 부분 | 제한적 | 5%+ | 엔터프라이즈 성향 |
| Gumroad | ✅ | ✅ | ✅ | 10%+ | 가장 간단, 수수료 높음 |
| Stripe (직접) | ✅ | ❌(직접 구현) | ❌(백엔드 필요) | 2.9%+ | MoR 아님, 세금 직접 |
| 국내 PortOne(아임포트)/부트페이 | ✅ | ❌ | ❌(백엔드 필요) | 낮음 | 자동화하려면 서버 필수 |

**결론: Lemon Squeezy로 시작.** 백엔드 없이 자동화가 되는 유일한 조합(구독+키+클라이언트 검증)이고, 나중에 국내 결제 비중이 커지면 PortOne+서버리스로 이전.

> 참고: Lemon Squeezy의 라이선스 키는 "문자열 검증"이라 기기 제한·공유 방지는 약합니다.
> 소액 구독에선 충분합니다. 공유가 문제될 규모가 되면 활성화 기기 수 제한(License API의 activation limit) + 서버리스 검증으로 강화.

## 3. 자동화 플로우

### (A) 일반 구독
```
사용자: 앱에서 "구독하기" 클릭
   ↓  (Lemon Squeezy Checkout 오버레이 — lemon.js 한 줄)
결제 완료
   ↓  Lemon Squeezy가 라이선스 키 자동 생성 + 이메일 발송 (운영자 개입 0)
사용자: 앱에 키 입력
   ↓  앱이 https://api.lemonsqueezy.com/v1/licenses/validate 로 POST (브라우저에서 직접)
검증 성공 → localStorage 저장 → 잠금 해제
```
- 구독이 활성인 동안 키가 유효, 해지되면 키 만료(License API가 상태 반영).
- 운영자가 할 일: **0** (환불·갱신도 Lemon Squeezy가 처리).

### (B) 연간 할인
별도 절차 없이 **연간 구독 상품 자체를 할인가로** 만듭니다(월 대비 33~34% 저렴).
Lemon Squeezy에서 월간/연간을 같은 상품의 두 variant로 두면, 체크아웃에서 사용자가 선택합니다.
→ 운영자 개입 0.

## 4. 가격 (시장 기반, 유저 획득 가능 구간)

벤치마크: 스픽 연 129,000(범용 영어) · 롱블랙 월 5,900 · 퍼블리 월 11,000(지식구독).
포지션: "폭은 좁지만 뾰족한" 스타트업 HR 특화 → 범용보다 낮게, 초기 획득 가능 구간.

| 서비스 | 월 | 연(할인) |
|---|---|---|
| HR English Coach Pro | 9,900 | 79,000 (33%↓) |
| 독서 큐레이션 멤버십 | 4,900 | 39,000 (34%↓) |
| (선택) 번들: 둘 다 | 12,900 | 99,000 |

> HR 실무자 명함 인증 할인은 제외했습니다. HR English Coach는 타깃 자체가 HR이라
> "HR이면 할인"이 변별력이 없고, 두 서비스 모두 **연간 구독 할인(33~34%)**으로 정책을 단순화합니다.

- 첫 100명은 **얼리버드 평생 할인**(예: 영구 월 6,900)으로 초기 리텐션 확보 권장 — 롱블랙이 쓴 전술.
- 무료 티어는 유지(HR 3케이스 / 책 4권): 검색 유입 → 무료 체험 → 전환 퍼널.

## 5. 단계별 도입 (Rollout)

**Phase 0 — 지금(수동, 완료됨)**
mailto 신청 + 수동 코드. 유료 사용자 ~20명까지.

**Phase 1 — 반자동 (반나절 세팅, 권장 다음 단계)**
1. Lemon Squeezy 가입 → Store 개설 → 각 서비스를 **Subscription Product**로 생성(월/연 variant), 라이선스 키 활성화
2. 발급된 월간/연간 체크아웃 URL 2개를 앱 `PAY_CONFIG`에 입력 → 배포
→ 이 시점부터 결제·발급·검증이 **완전 자동**. 운영자 개입 0.

**Phase 2 — 규모화 (유료 100명+ 또는 공유 남용 시)**
- 활성화 기기 수 제한(License API activation)
- 국내 결제 비중↑ 시 PortOne + Cloudflare Workers 라이선스 서버로 이전

## 6. 코드 연결 지점 (개발자용)

각 앱은 상단 **CONFIG 블록**의 값만 채우면 결제/검증이 켜집니다. 값이 비어 있으면 기존 mailto+오프라인 코드로 동작(안전한 폴백).

```js
const PAY_CONFIG = {
  // Lemon Squeezy에서 발급받아 채우기 (비우면 mailto 폴백)
  checkoutMonthly: "",         // 월간 구독 체크아웃 URL
  checkoutYearly: "",          // 연간 구독 체크아웃 URL
  licenseMode: "offline",      // "offline"(현재) | "lemonsqueezy"(전환 시)
};
```

라이선스 검증(전환 시): `POST https://api.lemonsqueezy.com/v1/licenses/validate`
body `license_key=<사용자입력>` → 응답 `valid: true && meta.subscription 활성` 이면 잠금 해제.
(구현 스텁은 `reading.html`의 `validateLicense()` 참고 — offline/lemonsqueezy 두 모드 지원.)

## 7. 예상 비용 · 손익 감

- Lemon Squeezy 수수료 ≈ 매출의 5% + 결제수수료(합계 대략 6~8%)
- 월 4,900 구독 100명 = 월 490,000 매출, 수수료 제외 대략 45만원 수준
- 고정비 0(정적 호스팅 무료, Tally 무료 티어). **손익분기가 매우 낮아 초기 실험에 적합.**
- AI 첨삭(Claude API)은 사용자 자기 키 또는 Pro 원가에 반영 — 현재 사용자 키 방식이라 운영자 부담 0.

---

## 8. 리스크 · 정직한 한계

- 클라이언트 검증은 우회 가능(개발자 도구). 소액·선의 기반 시장에선 실질 문제 낮음. 규모 시 Phase 2.
- Lemon Squeezy는 해외 서비스라 결제 화면·영수증이 영문/USD일 수 있음 → 국내 사용자 UX 확인 후, 필요시 원화 표기 안내 병기.
